## 修复总结

**问题**：用户登出后重新登录另一个账号时，在数据加载完成前会短暂看到上一个用户的交易记录和账户信息。

**原因**：登出时只清空了认证信息（token 和 user），但业务数据（账户、交易、分类、统计等）仍保留在 Pinia store 中。

**解决方案**：

### 1. 在各业务 store 中添加 `clear()` 方法

- **account.js:37-40** - 清空账户列表和加载状态
- **transaction.js:50-54** - 清空交易记录、分页信息和加载状态  
- **category.js:48-53** - 清空分类列表和加载状态
- **statistics.js:39-44** - 清空统计数据和加载状态

### 2. 在登出时调用所有 clear 方法

- **MainLayout.vue:82-89** - 用户主动点击退出登录时，先清空所有业务 store，再清除认证信息，最后跳转到登录页

### 3. Token 过期时也清空数据

- **api/index.js:17-29** - 添加 `clearAllStores()` 函数，当收到 401 未授权响应时（token 过期），清空所有业务数据后跳转登录页

这样确保了无论用户是主动登出还是 token 被动过期，所有上一个用户的业务数据都会被立即清空，新用户登录时不会看到旧数据。
